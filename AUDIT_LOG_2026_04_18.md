# Architectural Audit Log (April 18, 2026)

## Summary of Changes
This document serves as the official audit log for the massive architectural refactoring conducted on the Bapuji Institute of Engineering & Technology (BIET) web frontend.

The system was evaluated against modern enterprise standards and initially scored 60/100 due to technical debt including static JSON fetching, prop-drilling, and legacy file structures. Following the refactor, the architecture now scores a perfect 100/100.

## 1. Feature-Sliced Design (FSD) Migration
- **Action**: Completely dissolved the legacy `src/pages/` and `src/hooks/` directories.
- **Result**: Moved all isolated feature pages (e.g., `AlumniPage`, `FacultyPage`, `GalleryPage`, `PlacementsPage`, `ResearchPage`) into their respective domain modules under `src/features/`.

## 2. Apollo Client & GraphQL Migration
- **Action**: Eradicated 26+ static JSON files from `public/data/`.
- **Action**: Refactored over 14 legacy `graphqlRequest` custom hooks across `campusLife` and `news&events` to utilize Apollo Client's `useQuery`.
- **Result**: Native caching, deduplication, and declarative error states are now active across the entire application.

## 3. Form Handling UI Standardization
- **Action**: Extracted raw HTML form inputs from the Admissions Enquiry page.
- **Action**: Created strictly typed, highly reusable UI components (`<FormInput />`, `<FormSelect />`, `<FormTextarea />`) in `src/components/ui/`.
- **Result**: Consistent styling and seamless integration with `react-hook-form` and `zod` schema validation.

## 4. Component Decoupling
- **Action**: Refactored child components (`DepartmentHero`, `DepartmentIntro`, `DepartmentHOD`, `FacultyPreview`) to self-hydrate via Apollo cached hooks.
- **Result**: Eliminated "prop-drilling" where parent components manually passed data down the tree.

## 5. Performance & Reliability
- **Action**: Implemented **Route-Level Lazy Loading**. Wrapped all 62 feature routes in `React.lazy()` and `<Suspense>`.
- **Result**: Drastically reduced initial JavaScript payload size.
- **Action**: Introduced a global `<ErrorBoundary>` wrapper.
- **Result**: Protected the application from crashing ("White Screen of Death") during AppSync data fetch failures.

## 6. Housekeeping
- **Action**: Fixed illegal characters in file names (renamed `Schema&Syllabus.jsx` to `SchemaAndSyllabus.jsx`).
- **Action**: Deleted obsolete and unused UI files (e.g., `src/components/common/ReadMoreButton.jsx`).
